## A. Threat model (what we must defend against)

**Primary failure modes**
1. **Unsupported claims (hallucinations):** model states facts not found in allowed sources.
2. **Citation laundering:** citations attached to claims that the cited text does not actually support.
3. **Overgeneralization:** source says X in a narrow context; answer implies X universally.
4. **Conflict handling:** sources disagree; model picks one without noting conflict or basis.
5. **Missing critical info:** question requires specifics (time, jurisdiction, version) not present; model guesses.
6. **Prompt injection/data exfiltration:** retrieved/passed docs contain instructions to ignore policy, reveal secrets, or fabricate citations.
7. **Stale/temporal issues:** answer depends on current state; corpus may be outdated.
8. **Tooling errors:** retrieval returns irrelevant chunks; chunk metadata/locators wrong; reranker missing.

- **Evidence-first:** every factual claim must be linked to evidence *or* explicitly labeled “not in sources.”
- **Separation of channels:** (a) *evidence-grounded answer* vs (b) optional *best-effort/unsourced* (permissive only).

## 2) Claim verification rubric (support labels + resolution rules)

### Atomic claim definition
A claim should be *narrow enough* that a single source quote can support it. Split on:
- multiple facts (“X and Y”) → two claims
- conditionals (“If A then B”) → separate A, B, and the implication if needed
- quantities + units + timeframe → ensure each is explicit

### Support statuses
**supported**
- Evidence **explicitly states** the claim, or the claim is a **tight paraphrase** with no added constraints.
- Numbers/dates/thresholds must match.
- Scope matches (same product, version, geography, time).

**partially_supported**
- Evidence supports **part** of the claim but not all required details, e.g.:
  - source says “usually” and claim says “always”
  - source gives policy but not effective date; claim includes a date
  - evidence indicates “may” and claim implies certainty
- Action: in strict mode, either (a) narrow the claim to the supported portion, or (b) move to “what I don’t know.”

**unsupported**
- No allowed evidence entails the claim.
- Evidence is tangential, or only implies via guesswork.
- Action: remove claim (strict), or label as “not in sources” (permissive if allowed).

**contradictory**
- At least two allowed sources disagree materially on the claim.
- Action:
  - Prefer higher-authority source if a ranking policy exists (e.g., official policy > forum; newer > older if dated).
  - Otherwise present both and state conflict; avoid definitive conclusion.
  - In strict mode, if the conflict affects the user’s decision/action, ask for clarification or defer.

**opinion**
- Value judgments/recommendations not presented as fact.
- Ensure you don’t smuggle facts inside an opinion (“I think X is illegal” is factual).

### “Entailment” bar (practical test)
A claim is supported only if a reasonable auditor, reading the quoted snippet(s), would say:
- “Yes, that text clearly backs this statement,”
- without needing extra unstated assumptions.

### Citation laundering prevention
For each claim:
- require evidence quote(s) whose words overlap the claim’s key terms (entity, policy, metric) or provide a clear definitional mapping.
- block “nearby citations”: don’t cite a chunk that merely mentions the topic.

### Strict-mode gating rule (suggested)
- If any **required** claim is not `supported`, set `final.status` to:
  - `needs_clarification` if missing slots can resolve it, else
  - `insufficient_evidence`.
- Never output the unsupported claim as fact in `answer_markdown`.

```python
        "question": question,
        "passed_docs": passed_docs,
        "config": config,
        "retrieved_chunks": []
    }

        g_input = build_generator_input(state)
        g_out = run_generator(g_input)  # returns JSON

        # 2) Non-LLM lint (fast fail)
        lint_errors = run_lints(g_out)
        if lint_errors:
            # Turn lint errors into a patch_plan-like constraint for G
            patch = lint_errors_to_patch_plan(lint_errors)
            state["patch_plan"] = patch
            continue

        # 3) Build verifier working set (anti cherry-picking)
        working_set = build_working_set_texts(
            sources=g_out["sources"],
            passed_docs=state["passed_docs"],
            retrieved_chunks=state["retrieved_chunks"]
        )

        if missing_any_cited_source_text(g_out, working_set):
            # blocker: we cannot verify; either fetch missing chunk text or fail closed
            fetched = try_fetch_missing_texts(g_out["sources"])
            if not fetched:
                return fail_closed_insufficient_verification(g_out)

        v_input = {
            "generator_output": g_out,
            "verifier_input": {"working_set": working_set}
        }

        # 4) Run Verifier
        v_report = run_verifier(v_input)  # returns report JSON

        if v_report["verifier"]["verdict"] == "approve":
            # 5) Render final markdown in-app (authoritative = segments)
            final_md = render_markdown_from_segments(g_out["final"]["answer_segments"])
            g_out["final"]["answer_markdown"] = final_md  # optional
            return g_out

        # 6) Apply repair instructions
        patch_plan = v_report["verifier"].get("patch_plan", [])
        state["patch_plan"] = patch_plan

        # 7) If verifier suggests retrieval and config allows, orchestrator retrieves now
        for op in patch_plan:
            if op["op"] == "suggest_retrieval" and retrieval_allowed(config, state):
                chunks = retrieve(op["query"], k=op.get("k", config["retrieval"]["k"]),
                                  filters=op.get("filters", config["retrieval"]["filters"]))
                state["retrieved_chunks"].extend(dedupe(chunks))

        # loop continues: next iteration re-runs G with patch_plan + new evidence

    # If we exhausted iterations, fail closed with best available non-hallucinatory response:
    return {
        "final": {
            "status": "insufficient_evidence",
            "answer_segments": [
                {
                    "segment_id": "A1",
                    "segment_type": "warning",
                    "text": "I can’t safely produce a verified answer from the available sources.",
                    "claim_ids": [],
                    "citation_markers": []
                }
            ],
            "follow_up_questions": ["Can you provide the relevant document/policy text or allow corpus search?"]
        }
    }
```

Right now `request.passed_docs[*].source_id` and `sources[*].source_id` share a name but can mean different things (corpus doc IDs vs the short `S1` label used in citations). That’s a common footgun.

**Recommendation (v1.1 but easy now):**
- Keep `sources[*].source_id` as the **citation label** (`S1`, `S2`, …).
- Add `sources[*].origin` describing where it came from:

```json
"sources": [{
  "source_id": "S1",
  "title": "...",
  "uri": null,
  "type": "passed_doc",
  "origin": { "passed_doc_id": "PD123", "chunk_id": null }
}]
```

For retrieved chunks:
```json
"origin": { "corpus_doc_id": "doc_7781", "chunk_id": "c12" }
```

Then in `request.passed_docs[]`, rename `source_id` → `passed_doc_id` (or allow both during migration).

This makes “phantom_source” detection unambiguous.

---

### B. Enforce segment requirements with `if/then` (deterministic, no heuristics)
You already locked: citations required iff `segment_type="answer"` and `citation_style!="none"`.
Schema can at least enforce the `segment_type="answer"` half:

```json
"allOf": [
  {
    "if": { "properties": { "segment_type": { "const": "answer" } } },
    "then": {
      "properties": {
        "claim_ids": { "type": "array", "minItems": 1 },
        "citation_markers": { "type": "array", "minItems": 1 }
      }
    }
  }
]
```

Put that inside the `answer_segments.items` schema.

You *can’t* (cleanly) express “only if citation_style != none” inside the segment item because JSON Schema doesn’t naturally reference sibling paths without `$data` extensions—so keep the second half as a linter rule.

### C. Validate `citation_markers` format with a regex pattern
Add this to `citation_markers.items`:

- permissive but strict enough:

Example patch:

Your linter/V will still check the `source_id` exists in `sources[]`.

### Python
```python
import hashlib, json, re

def canon_json(obj) -> bytes:
    return json.dumps(obj, sort_keys=True, separators=(",", ":"), ensure_ascii=False).encode("utf-8")

def norm_text_lf(s: str) -> str:
    return s.replace("\r\n", "\n").replace("\r", "\n")

def sha256_hex(b: bytes) -> str:
    return hashlib.sha256(b).hexdigest()

def text_hash(text: str, norm: str = "lf") -> str:
    if norm == "lf":
        text = norm_text_lf(text)
    elif norm == "raw":
        pass
    else:
        raise ValueError("unknown norm")
    return "sha256:" + sha256_hex(text.encode("utf-8"))

def build_working_set_manifest(working_set: list, text_norm="lf") -> list:
    # Deterministic ordering: by source_id (since it's S1,S2,...)
    ws_sorted = sorted(working_set, key=lambda x: x["source_id"])
    manifest = []
    for w in ws_sorted:
        manifest.append({
            "source_id": w["source_id"],
            "type": w["type"],
            "origin": w["origin"],
            "title": w.get("title"),
            "uri": w.get("uri"),
            "chunk_id": w.get("chunk_id"),
            "locators": w.get("locators", {}),
            "text_hash": text_hash(w["text"], norm=text_norm),
            "text_hash_norm": text_norm
        })
    return manifest

def select_config_for_snapshot(config: dict) -> dict:
    return {
        "mode": config["mode"],
        "allow_general_knowledge": config["allow_general_knowledge"],
        "citation_style": config["citation_style"],
        "quote_policy": config["quote_policy"],
        "high_risk_policy": config["high_risk_policy"],
        "retrieval": {
            "enabled": config["retrieval"]["enabled"],
            "k": config["retrieval"]["k"],
            "filters": config["retrieval"]["filters"]
        }
    }

def build_snapshot(question: str, config: dict, routing: dict, working_set: list) -> dict:
    return {
        "question": question,
        "config_selected": select_config_for_snapshot(config),
        "orchestrator_routing": routing,
        "working_set_manifest": build_working_set_manifest(working_set, text_norm="lf")
    }

def snapshot_hash(snapshot_obj: dict) -> str:
    return "sha256:" + sha256_hex(canon_json(snapshot_obj))
```

### TypeScript (Node)
```ts
import crypto from "crypto";

function canonJson(obj: any): string {
  // Stable stringify: simplest is a library (json-stable-stringify).
  // If rolling your own, ensure recursive key sort.
  const stableStringify = require("json-stable-stringify");
  return stableStringify(obj, { space: "" });
}

function normTextLF(s: string): string {
  return s.replace(/\r\n/g, "\n").replace(/\r/g, "\n");
}

function sha256Hex(s: string): string {
  return crypto.createHash("sha256").update(s, "utf8").digest("hex");
}

function textHash(text: string, norm: "lf" | "raw" = "lf"): string {
  const t = norm === "lf" ? normTextLF(text) : text;
  return `sha256:${sha256Hex(t)}`;
}

function buildWorkingSetManifest(workingSet: any[], norm: "lf" | "raw" = "lf") {
  const ws = [...workingSet].sort((a,b) => a.source_id.localeCompare(b.source_id));
  return ws.map(w => ({
    source_id: w.source_id,
    type: w.type,
    origin: w.origin,
    title: w.title ?? null,
    uri: w.uri ?? null,
    chunk_id: w.chunk_id ?? null,
    locators: w.locators ?? {},
    text_hash: textHash(w.text, norm),
    text_hash_norm: norm
  }));
}

function snapshotHash(snapshotObj: any): string {
  return `sha256:${sha256Hex(canonJson(snapshotObj))}`;
}
```

### 3) Prevent duplicate IDs in arrays (cheap hygiene)

- `claims[*].evidence_ids`: `uniqueItems: true`

## TypeScript lint skeleton (v1.1)

```ts
// Lint severity levels
type Severity = "blocker" | "warning";

type LintIssue = {
  type: string;
  severity: Severity;
  scope: "global" | "tooling" | "sources" | "evidence" | "claims" | "segments" | "provenance" | "routing";
  details: string;
  location?: string;
  related_ids?: Partial<{ source_id: string; evidence_id: string; claim_id: string; segment_id: string }>;
};

type LintResult = { ok: boolean; issues: LintIssue[] };

function canonJson(x: unknown): string {
  // Use a stable stringify library in production.
  // e.g. json-stable-stringify or fast-json-stable-stringify
  const stableStringify = require("fast-json-stable-stringify");
  return stableStringify(x);
}

function fail(issues: LintIssue[], issue: LintIssue) {
  issues.push(issue);
}

function isBlocker(issues: LintIssue[]) {
  return issues.some(i => i.severity === "blocker");
}

function assertUniqueBy<T>(arr: T[], keyFn: (t: T) => string, issueType: string, scope: LintIssue["scope"], issues: LintIssue[]) {
  const seen = new Set<string>();
  for (const item of arr) {
    const k = keyFn(item);
    if (seen.has(k)) {
      fail(issues, { type: issueType, severity: "blocker", scope, details: `Duplicate key: ${k}` });
    }
    seen.add(k);
  }
}

function parseCitationSid(marker: string): string | null {
  // marker already schema-validated to ^\[S[1-9][0-9]*(?:#...)?\]$
  const m = marker.match(/^\[(S[1-9][0-9]*)(?:#[^\]]+)?\]$/);
  return m ? m[1] : null;
}

// ------------ SCHEMA VALIDATION (Ajv) ------------
function validateSchemaOrThrow(_schemaId: string, _data: any): LintIssue[] {
  // Replace with Ajv calls. Return list of schema issues as blockers.
  return [];
}

// ------------ MAIN LINT ENTRYPOINT ------------
export function lintAll(genIn: any, genOut: any): LintResult {
  const issues: LintIssue[] = [];

  // 1) Schema validation
  issues.push(...validateSchemaOrThrow("schemas/generator_input.v1.1.json", genIn));
  issues.push(...validateSchemaOrThrow("schemas/generator_output.v1.1.json", genOut));
  if (isBlocker(issues)) return { ok: false, issues };

  // 2) Input → Output binding lints (need both in+out)
  lintRoutingVerbatim(genIn, genOut, issues);
  lintProvenanceVerbatim(genIn, genOut, issues);
  lintWorkingSetSourcesAlignment(genIn, genOut, issues);
  lintContentHashAlignment(genIn, genOut, issues);

  // 3) Cross-field output lints (output only, plus config/routing inside output)
  lintCitationConditionality(genOut, issues);
  lintBestEffortEligibility(genOut, issues);
  lintForcedStrictOverride(genOut, issues);
  lintIdIntegrity(genOut, issues);
  lintToolLogConsistency(genOut, issues);
  lintUniquenessById(genOut, issues);

  return { ok: !isBlocker(issues), issues };
}

// ------------ 2) INPUT → OUTPUT BINDING ------------
function lintRoutingVerbatim(genIn: any, genOut: any, issues: LintIssue[]) {
  if (genOut.decision?.routing_source !== "orchestrator") {
    fail(issues, { type: "routing_source_not_orchestrator", severity: "blocker", scope: "routing",
      details: `Expected decision.routing_source="orchestrator".` });
    return;
  }
  const a = canonJson(genOut.decision?.routing);
  const b = canonJson(genIn.orchestrator_routing);
  if (a !== b) {
    fail(issues, { type: "routing_not_copied_verbatim", severity: "blocker", scope: "routing",
      details: "decision.routing must equal orchestrator_routing (canonical JSON compare)." });
  }
}

function lintProvenanceVerbatim(genIn: any, genOut: any, issues: LintIssue[]) {
  const pin = genIn.provenance_input;
  const pout = genOut.provenance;

  const fields: Array<[string, string]> = [
    ["generator_input_hash_alg", "generator_input_hash_alg"],
    ["generator_input_hash", "generator_input_hash"],
    ["generator_input_hash_scope", "generator_input_hash_scope"],
    ["orchestrator_run_id", "orchestrator_run_id"]
  ];

  for (const [inKey, outKey] of fields) {
    if (pin?.[inKey] !== pout?.[outKey]) {
      fail(issues, {
        type: "provenance_not_copied_verbatim",
        severity: "blocker",
        scope: "provenance",
        details: `provenance.${outKey} must equal provenance_input.${inKey}`
      });
      break;
    }
  }
}

function lintWorkingSetSourcesAlignment(genIn: any, genOut: any, issues: LintIssue[]) {
  const ws: any[] = genIn.working_set ?? [];
  const sources: any[] = genOut.sources ?? [];

  // 1:1 mapping by source_id
  const wsBySid = new Map<string, any>();
  for (const w of ws) wsBySid.set(w.source_id, w);

  for (const s of sources) {
    const w = wsBySid.get(s.source_id);
    if (!w) {
      fail(issues, {
        type: "source_not_in_working_set",
        severity: "blocker",
        scope: "sources",
        details: `sources includes ${s.source_id} not present in generator_input.working_set`,
        related_ids: { source_id: s.source_id }
      });
      continue;
    }
    // origin must match exactly (canonical compare)
    if (canonJson(s.origin) !== canonJson(w.origin)) {
      fail(issues, {
        type: "source_origin_mismatch",
        severity: "blocker",
        scope: "sources",
        details: `origin mismatch for ${s.source_id}`,
        related_ids: { source_id: s.source_id }
      });
    }
  }

  // Ensure no missing sources for working_set (optional but recommended)
  const sourceSidSet = new Set(sources.map(s => s.source_id));
  for (const w of ws) {
    if (!sourceSidSet.has(w.source_id)) {
      fail(issues, {
        type: "working_set_item_missing_in_sources",
        severity: "warning",
        scope: "sources",
        details: `working_set contains ${w.source_id} but sources[] does not list it (ok only if unused).`,
        related_ids: { source_id: w.source_id }
      });
    }
  }
}

function lintContentHashAlignment(genIn: any, genOut: any, issues: LintIssue[]) {
  // If sources[*].content_hash present, ensure it matches provenance_input.working_set_manifest text_hash for same source_id+origin
  const manifest: any[] = genIn.provenance_input?.working_set_manifest ?? [];
  const manKey = (m: any) => `${m.source_id}::${canonJson(m.origin)}`;
  const manByKey = new Map(manifest.map(m => [manKey(m), m]));

  for (const s of (genOut.sources ?? [])) {
    if (!s.content_hash) continue;
    const m = manByKey.get(`${s.source_id}::${canonJson(s.origin)}`);
    if (!m) {
      fail(issues, {
        type: "content_hash_manifest_missing",
        severity: "blocker",
        scope: "provenance",
        details: `No working_set_manifest entry for ${s.source_id} with matching origin.`,
        related_ids: { source_id: s.source_id }
      });
      continue;
    }
    if (s.content_hash !== m.text_hash) {
      fail(issues, {
        type: "content_hash_mismatch",
        severity: "blocker",
        scope: "provenance",
        details: `sources.content_hash != working_set_manifest.text_hash for ${s.source_id}`,
        related_ids: { source_id: s.source_id }
      });
    }
  }
}

// ------------ 3) CROSS-FIELD OUTPUT LINTS ------------
function lintCitationConditionality(genOut: any, issues: LintIssue[]) {
  const style = genOut.request?.config?.citation_style;
  const segs: any[] = genOut.final?.answer_segments ?? [];

  for (const seg of segs) {
    if (seg.segment_type !== "answer") continue;
    const n = (seg.citation_markers ?? []).length;

    if (style === "none" && n > 0) {
      fail(issues, {
        type: "citations_present_when_disabled",
        severity: "blocker",
        scope: "segments",
        details: `citation_style="none" but answer segment has citation_markers.`,
        related_ids: { segment_id: seg.segment_id }
      });
    }
    if (style !== "none" && n === 0) {
      fail(issues, {
        type: "missing_citations_in_answer_segment",
        severity: "blocker",
        scope: "segments",
        details: `citation_style!=none but answer segment lacks citation_markers.`,
        related_ids: { segment_id: seg.segment_id }
      });
    }

    // Phantom source check
    const sourceIds = new Set((genOut.sources ?? []).map((s: any) => s.source_id));
    for (const m of (seg.citation_markers ?? [])) {
      const sid = parseCitationSid(m);
      if (sid && !sourceIds.has(sid)) {
        fail(issues, {
          type: "phantom_citation_source",
          severity: "blocker",
          scope: "segments",
          details: `Citation marker references missing source_id ${sid}`,
          related_ids: { segment_id: seg.segment_id, source_id: sid }
        });
      }
    }
  }
}

function lintBestEffortEligibility(genOut: any, issues: LintIssue[]) {
  const mode = genOut.request?.config?.mode;
  const allowGK = genOut.request?.config?.allow_general_knowledge;
  const segs: any[] = genOut.final?.answer_segments ?? [];

  for (const seg of segs) {
    if (seg.segment_type !== "best_effort_unsourced") continue;
    if (mode !== "permissive" || allowGK !== true) {
      fail(issues, {
        type: "best_effort_not_allowed",
        severity: "blocker",
        scope: "segments",
        details: "best_effort_unsourced requires permissive mode and allow_general_knowledge=true",
        related_ids: { segment_id: seg.segment_id }
      });
    }
  }
}

function lintForcedStrictOverride(genOut: any, issues: LintIssue[]) {
  const r = genOut.decision?.routing;
  const forced = genOut.decision?.risk_level === "high"
    && r?.actionable === true
    && r?.forced_behavior?.force_strict_for_actionable === true;

  if (!forced) return;
  if (genOut.final?.status !== "answer") return;

  const claimById = new Map<string, any>((genOut.claims ?? []).map((c: any) => [c.claim_id, c]));
  for (const seg of (genOut.final?.answer_segments ?? [])) {
    if (seg.segment_type !== "answer") continue;
    for (const cid of (seg.claim_ids ?? [])) {
      const c = claimById.get(cid);
      if (!c) continue;
      if (c.support_status !== "supported") {
        fail(issues, {
          type: "forced_strict_violation_high_risk_actionable",
          severity: "blocker",
          scope: "claims",
          details: `High-risk forced strict: claim ${cid} is ${c.support_status}`,
          related_ids: { segment_id: seg.segment_id, claim_id: cid }
        });
      }
    }
  }
}

function lintIdIntegrity(genOut: any, issues: LintIssue[]) {
  const claimIds = new Set((genOut.claims ?? []).map((c: any) => c.claim_id));
  const evidenceIds = new Set((genOut.evidence ?? []).map((e: any) => e.evidence_id));
  const sourceIds = new Set((genOut.sources ?? []).map((s: any) => s.source_id));

  for (const seg of (genOut.final?.answer_segments ?? [])) {
    for (const cid of (seg.claim_ids ?? [])) {
      if (!claimIds.has(cid)) {
        fail(issues, { type: "id_integrity_violation", severity: "blocker", scope: "segments",
          details: `segment references missing claim_id ${cid}`, related_ids: { segment_id: seg.segment_id, claim_id: cid }});
      }
    }
  }

  for (const c of (genOut.claims ?? [])) {
    for (const eid of (c.evidence_ids ?? [])) {
      if (!evidenceIds.has(eid)) {
        fail(issues, { type: "id_integrity_violation", severity: "blocker", scope: "claims",
          details: `claim references missing evidence_id ${eid}`, related_ids: { claim_id: c.claim_id, evidence_id: eid }});
      }
    }
  }

  for (const e of (genOut.evidence ?? [])) {
    if (!sourceIds.has(e.source_id)) {
      fail(issues, { type: "id_integrity_violation", severity: "blocker", scope: "evidence",
        details: `evidence references missing source_id ${e.source_id}`, related_ids: { evidence_id: e.evidence_id, source_id: e.source_id }});
    }
  }
}

function lintToolLogConsistency(genOut: any, issues: LintIssue[]) {
  const used = genOut.decision?.retrieval_used;
  const calls = genOut.tool_log?.retrieval_calls ?? [];
  if (used === true && calls.length === 0) {
    fail(issues, { type: "tool_log_inconsistent", severity: "blocker", scope: "tooling",
      details: "retrieval_used=true but retrieval_calls is empty" });
  }
  if (used === false && calls.length > 0) {
    fail(issues, { type: "tool_log_inconsistent", severity: "blocker", scope: "tooling",
      details: "retrieval_used=false but retrieval_calls is non-empty" });
  }
```

```ts
type SecretHit = {
  kind: string;
  match: string;
  index: number;
};

const SECRET_PATTERNS: Array<{ kind: string; re: RegExp }> = [
  // Private keys
  { kind: "private_key_pem", re: /-----BEGIN (?:RSA |EC |DSA |OPENSSH )?PRIVATE KEY-----/g },

  // JWT (very common)
  { kind: "jwt", re: /\beyJ[A-Za-z0-9_-]{10,}\.[A-Za-z0-9_-]{10,}\.[A-Za-z0-9_-]{10,}\b/g },

  // AWS access key id
  { kind: "aws_access_key_id", re: /\bAKIA[0-9A-Z]{16}\b/g },
  // AWS secret-ish (high false positives; keep conservative)
  { kind: "aws_secret_key", re: /\b(?:AWS|aws).{0,20}(?:secret|Secret).{0,20}([A-Za-z0-9/+=]{40})\b/g },

  // Slack tokens (legacy + modern-ish)
  { kind: "slack_token", re: /\bxox[baprs]-[A-Za-z0-9-]{10,}\b/g },

  // GitHub tokens (classic + fine-grained prefixes)
  { kind: "github_token", re: /\bgh[pousr]_[A-Za-z0-9]{20,}\b/g },

  // Stripe keys
  { kind: "stripe_key", re: /\bsk_(live|test)_[A-Za-z0-9]{16,}\b/g },

  // Generic "api key" assignment (conservative)
  { kind: "api_key_assignment", re: /\b(api[_-]?key|token|secret)\b\s*[:=]\s*[A-Za-z0-9_\-./+=]{12,}/gi },

  // Email (PII-ish; decide whether to block or warn in your policy)
  { kind: "email", re: /\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}\b/g },

  // US SSN (PII; optional)
  { kind: "ssn_us", re: /\b\d{3}-\d{2}-\d{4}\b/g }
];

function scanSecrets(text: string): SecretHit[] {
  const hits: SecretHit[] = [];
  for (const { kind, re } of SECRET_PATTERNS) {
    re.lastIndex = 0;
    let m: RegExpExecArray | null;
    while ((m = re.exec(text)) !== null) {
      hits.push({ kind, match: m[0], index: m.index });
      // safety: avoid infinite loops on zero-width matches
      if (m.index === re.lastIndex) re.lastIndex++;
    }
  }
  return hits;
}

function summarizeHits(hits: SecretHit[]): string {
  // Don’t log full secrets; show kind + a shortened snippet
  const uniq = new Map<string, SecretHit>();
  for (const h of hits) uniq.set(`${h.kind}:${h.match.slice(0, 8)}`, h);
  return [...uniq.values()]
    .slice(0, 5)
    .map(h => `${h.kind} @${h.index} "${h.match.slice(0, 12)}…"`)
    .join(", ");
}

export function lintSecrets(genOut: any, issues: LintIssue[]) {
  // 1) Answer segments: blocker on any hit
  for (const seg of (genOut.final?.answer_segments ?? [])) {
    const text = String(seg.text ?? "");
    const hits = scanSecrets(text);
    if (hits.length > 0) {
      fail(issues, {
        type: "secrets_or_pii_in_answer_segment",
        severity: "blocker",
        scope: "segments",
        details: `Detected potential secrets/PII in answer segment. Hits: ${summarizeHits(hits)}`,
        related_ids: { segment_id: seg.segment_id }
      });
    }
  }

  // 2) Evidence quotes: blocker unless redacted
  for (const ev of (genOut.evidence ?? [])) {
    const quote = String(ev.quote ?? "");
    const hits = scanSecrets(quote);
    if (hits.length === 0) continue;

    const redacted = ev.quote_redacted === true;
    if (!redacted) {
      fail(issues, {
        type: "secrets_or_pii_in_evidence_quote_unredacted",
        severity: "blocker",
        scope: "evidence",
        details: `Detected potential secrets/PII in evidence quote and quote_redacted!=true. Hits: ${summarizeHits(hits)}`,
        related_ids: { evidence_id: ev.evidence_id, source_id: ev.source_id }
      });
    }
  }
}
```

## 2) Evidence quote substring precheck (verbatim enforcement)

This is a deterministic guardrail matching the spec: evidence quotes must be verbatim substrings of `working_set[].text` for the same `source_id` (after LF normalization).

```ts
function normLF(s: string): string {
  return s.replace(/\r\n/g, "\n").replace(/\r/g, "\n");
}

export function lintEvidenceQuotesAreSubstrings(genIn: any, genOut: any, issues: LintIssue[]) {
  const wsBySid = new Map<string, any>();
  for (const w of (genIn.working_set ?? [])) wsBySid.set(w.source_id, w);

  for (const ev of (genOut.evidence ?? [])) {
    const sid = ev.source_id;
    const w = wsBySid.get(sid);
    if (!w) {
      fail(issues, {
        type: "evidence_source_not_in_working_set",
        severity: "blocker",
        scope: "evidence",
        details: `Evidence references source_id ${sid} not present in generator_input.working_set`,
        related_ids: { evidence_id: ev.evidence_id, source_id: sid }
      });
      continue;
    }

    // Optional: chunk_id coherence check if you want it here (schema covers some, but not input-output)
    if (ev.chunk_id && w.chunk_id && ev.chunk_id !== w.chunk_id) {
      fail(issues, {
        type: "evidence_chunk_id_mismatch",
        severity: "blocker",
        scope: "evidence",
        details: `Evidence chunk_id ${ev.chunk_id} does not match working_set chunk_id ${w.chunk_id} for ${sid}`,
        related_ids: { evidence_id: ev.evidence_id, source_id: sid }
      });
    }

    const hay = normLF(String(w.text ?? ""));
    const needle = normLF(String(ev.quote ?? ""));

    if (!needle || hay.indexOf(needle) === -1) {
      fail(issues, {
        type: "evidence_quote_not_in_working_set_text",
        severity: "blocker",
        scope: "evidence",
        details: `Evidence quote is not a verbatim substring of working_set text for ${sid} (after LF normalization).`,
        related_ids: { evidence_id: ev.evidence_id, source_id: sid }
      });
    }
  }
}
```
